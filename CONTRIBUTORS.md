# Contributing to CodeRLM

CodeRLM is a one-person project right now, and I'm keeping the code itself closed to outside contributions while the design is in flight. 

The server, plugin, and workflows that I'm wanting to support long term all need to evolve together and I haven't yet figured out a contribution model that works.

Sorry, I know it's frustrating if you were hoping to add a language or new functionality that is working for you.

That said, there's plenty of ways you can help
- Bugs. If something doesn't work the way the docs describe, open an issue with reproduction steps, the language(s) involved, and any relevant logs. These I'll dig into quickly.
- Feature ideas. New language support, API additions, workflow improvements file an issue and tell me about what you're thinking.
- Questions. Issues are also fine for "how does X work" or "is Y the intended behavior."

Forks are welcome. If you build something interesting on top of CodeRLM I'd be excited to see and learn about!
